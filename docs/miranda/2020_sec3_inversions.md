# Sec 3: 転倒数ベースGreedy（符号なし順列）

対象問題：**SbR, SbT, SbRT**（符号なし）

---

## 前提定義

### 順列と単位順列

長さ n の **符号なし順列** π = (π₁ π₂ … πₙ) は {1, 2, …, n} の並び替えである。
**単位順列** ι = (1 2 … n) がソートの目標となる。

### 再配置の種類と長さ

**逆位（reversal）** ρ(i, j)（1 ≤ i < j ≤ n）は区間 [i, j] の要素を逆順にする：

(π₁ … πᵢ₋₁  πᵢ … πⱼ  πⱼ₊₁ … πₙ) · ρ(i,j) = (π₁ … πᵢ₋₁  πⱼ … πᵢ  πⱼ₊₁ … πₙ)

**転置（transposition）** τ(i, j, k)（1 ≤ i < j < k ≤ n+1）は隣接する2つの区間 [i, j) と [j, k) を交換する：

(π₁ … πᵢ₋₁  πᵢ … πⱼ₋₁  πⱼ … πₖ₋₁  πₖ … πₙ) · τ(i,j,k) = (π₁ … πᵢ₋₁  πⱼ … πₖ₋₁  πᵢ … πⱼ₋₁  πₖ … πₙ)

再配置 β の **長さ** |β| を影響を受ける要素数で定義する：

- |ρ(i, j)| = j − i + 1
- |τ(i, j, k)| = k − i

整数 λ > 1 に対し、|β| ≤ λ を満たす再配置を **λ-再配置** と呼ぶ。
**再配置モデル** M はソートに使用できる λ-再配置の集合（逆位のみ・転置のみ・両方など）を指す。

### コスト関数

再配置 β のコストを **f(β) = |β|** と定義する（本節では α = 1）。
操作列 S = β₁, β₂, …, βₘ の総コストは f(S) = Σ f(βᵢ)。

### λ-再配置距離

順列 π とモデル M に対する **λ-再配置距離** cᴹλ(π) は、M の λ-再配置のみを使って π を ι にソートする操作列の最小総コストである。
本節では以下の記法を使う：

- cʳλ(π)：SbR（符号なし λ-逆位のみ）の距離
- cᵗλ(π)：SbT（λ-転置のみ）の距離
- cʳᵗλ(π)：SbRT（λ-逆位と λ-転置）の距離

### 転倒数

要素の対 (πᵢ, πⱼ) が **転倒（inversion）** であるとは、i < j かつ πᵢ > πⱼ が成り立つことをいう。
**転倒数** Inv(π) を π に含まれる転倒の総数として定義する：

Inv(π) = |{(i, j) : i < j, πᵢ > πⱼ}|

**基本性質**：
- Inv(π) = 0 ⟺ π = ι（符号なし順列においては単位順列のみ）
- Inv(π) > 0 ならば隣接転倒 (πᵢ, πᵢ₊₁) が必ず存在する（Lemma 1）

**例**：π = (3 4 1 2) の転倒は (3,1), (3,2), (4,1), (4,2) の4つなので Inv(π) = 4。

また、π と再配置 β に対し **ΔInv(π, β) = Inv(π) − Inv(π·β)** を β による転倒数の削減量とする。

---

## アルゴリズム（Algorithm 1）

コスト1あたりに最も多くの転倒を除去できる λ-再配置を毎ステップ貪欲に選んで適用する。

```
入力：符号なし順列 π、整数 λ > 1、λ-再配置のみからなるモデル M
出力：π をソートする M の操作列

while Inv(π) > 0:
    β = argmax{ ΔInv(π, β) / f(β) : β ∈ M }   ← コスト効率最大の操作を選ぶ
    π ← π · β
    S に β を追加
return S
```

Inv(π) = 0 ⟺ π = ι（符号なし）なので、転倒数が0になれば終了する。
Lemma 5 より常に転倒数を減らす操作が存在するため、アルゴリズムは必ず終了する。

---

## 鍵となる補題

**Lemma 1**（隣接転倒の存在）
Inv(π) > 0 ならば、隣接転倒 (πᵢ, πᵢ₊₁) が必ず存在する。

証明の概略：|π₁| < … < |πᵢ| となる最長の先頭部分列を考えると、Inv(π) > 0 より i < n であり、(πᵢ, πᵢ₊₁) が転倒になる。

---

**Lemma 3**（各操作の比の上界）

任意の順列 π と整数 λ に対して：
- λ-逆位 ρ：ΔInv(π, ρ) / |ρ| ≤ (λ−1)/2
- λ-転置 τ：ΔInv(π, τ) / |τ| ≤ λ/4

証明の概略：長さ k の逆位が除去できる転倒は区間内の k(k−1)/2 個以下なので比 ≤ (k−1)/2。転置 τ(i,j,k) は区間 [i,j) と [j,k) をまたぐ転倒のみ除去できるため、その数は (j−i)(k−j) ≤ (k/2)² となり比 ≤ k/4。

---

**Lemma 4**（距離の下界）

任意の符号なし順列 π と λ > 1 に対して：
- cʳλ(π) ≥ cʳᵗλ(π) ≥ 2·Inv(π) / (λ−1)
- cᵗλ(π) ≥ 4·Inv(π) / λ

証明の概略：Lemma 3 より、転倒数を1減らす最小コストは逆位で 2/(λ−1)、転置で 4/λ。Inv(π) 個の転倒をすべて解消するには少なくともその積のコストが必要。

---

**Lemma 5**（アルゴリズムのコスト上界）

任意の符号なし順列 π と λ > 1 に対して：
- cᵗλ(π) ≤ 2·Inv(π)
- cʳᵗλ(π) ≤ cʳλ(π) ≤ 2·Inv(π)

証明の概略：Lemma 1 より隣接転倒 (πᵢ, πᵢ₊₁) が存在するとき、τ(i, i+1, i+2) と ρ(i, i+1) はいずれもコスト2でその転倒を1つ除去する。よって「コスト2で転倒1削減」が常に可能であり、Inv(π) 個の転倒を解消する総コストは 2·Inv(π) 以下。

---

## 主要定理

**Theorem 1**：λ > 1 のとき、Algorithm 1 は
- SbR と SbRT に対して：**(λ−1)-近似**
- SbT に対して：**(λ/2)-近似**

**証明の流れ**：

| | SbR / SbRT | SbT |
|---|---|---|
| アルゴリズムのコスト上界（Lemma 5） | ≤ 2·Inv(π) | ≤ 2·Inv(π) |
| 最適コストの下界（Lemma 4） | ≥ 2·Inv(π)/(λ−1) | ≥ 4·Inv(π)/λ |
| 近似係数 | 2·Inv(π) / [2·Inv(π)/(λ−1)] = **λ−1** | 2·Inv(π) / [4·Inv(π)/λ] = **λ/2** |
